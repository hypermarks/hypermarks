Okay, there are a couple solutions I can think of for getting the count. They both have pros and cons. I've put a code example of each in this gist.

1) Use snapshot.forEach()  -- I think this one is what you need

Pros: simple and easy to set up

Cons: gets slower as more records are added; I don't use this for anything that might contain more than several hundred records (maybe Firebase team will comment on their performance recommendations)

2) Use a counter and set it in a transaction

Pros: lightning fast to fetch the count

Cons: transactions are complex and easy to introduce bugs; have to remember to increment/decrement any time a tag is added/removed
